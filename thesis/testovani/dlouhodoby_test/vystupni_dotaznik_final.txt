# Výstupní dotazník — dlouhodobé testování aplikace Foody

**Aplikace:** Foody (diplomová práce, ČVUT FEL)
**Autor:** Bc. Jakub Sebastian Andráš
**Vedoucí práce:** Ing. Ivo Malý, Ph.D.

---

## Před vyplněním

Děkuji za to, že jsi aplikaci Foody dva týdny používal/a v běžném životě. Tento dotazník doplňuje data, která jsem získal automatickým exportem z aplikace, o tvůj subjektivní pohled. Bez něj by analýza nebyla úplná.

**Pár praktických informací:**

- Vyplnění zabere přibližně **10 minut**.
- Vyplň prosím **do 24 hodin** od posledního dne, kdy jsi aplikaci aktivně používal/a, dokud máš zážitek čerstvý.
- Odpovědi jsou anonymizovány — v textu diplomové práce budeš uveden/a pouze kódem (např. `LP1`), tvé jméno se v práci nikde neobjeví.
- U otevřených otázek na konci stačí **1 až 3 věty**. Krátká a věcná odpověď je pro mě cennější než dlouhá úvaha.

## Sekce A — System Usability Scale (SUS)

> **Co to je:** SUS je standardizovaný dotazník použitelnosti, který se používá v HCI od roku 1986. Skládá se z deseti tvrzení; u každého zatrhneš, jak moc s ním souhlasíš na škále 1 až 5. Liché položky jsou formulovány pozitivně, sudé negativně — to není chyba, je to záměr (vyvažuje to tendenci automaticky souhlasit).
>
> **Pokyn:** U každého tvrzení zatrhni, jak moc s ním souhlasíš, kde **1 = zcela nesouhlasím** a **5 = zcela souhlasím**. Vyhodnocení skóre udělám sám.

| # | Tvrzení | 1 | 2 | 3 | 4 | 5 |
|---|---|:-:|:-:|:-:|:-:|:-:|
| 1 | Myslím, že bych aplikaci chtěl/a používat často. |  |
| 2 | Aplikace mi přišla zbytečně složitá. |  |
| 3 | Aplikace se používala snadno. |  |
| 4 | Myslím, že bych potřeboval/a pomoc technika, abych aplikaci dokázal/a používat. |  |
| 5 | Jednotlivé funkce aplikace jsou dobře integrované. |  |
| 6 | V aplikaci je příliš mnoho nekonzistencí. |  |
| 7 | Většina lidí se podle mě naučí aplikaci používat velmi rychle. |  |
| 8 | Aplikace mi přišla nepřehledná a těžko ovladatelná. |  |
| 9 | Při používání aplikace jsem se cítil/a sebejistě. |  |
| 10 | Než jsem aplikaci dokázal/a používat, musel/a jsem se naučit hodně věcí. |  |

## Sekce B — kognitivní a časová zátěž (Raw NASA-TLX)

> **Co to je:** Zkrácená verze nástroje, který vyvinula NASA pro měření vnímané zátěže při práci s rozhraním. Měří, jak moc tě používání aplikace stálo úsilí, pozornosti a nervů — to je přesně to, co u dlouhodobého testování chci pochopit (na rozdíl od krátkodobých laboratorních testů, kde se měří hlavně to, jestli úloha jde dokončit).
>
> **Pokyn:** U každé dimenze napiš číslo od **0 do 20**, které nejlépe vystihuje, jak silně jsi ji pociťoval/a při běžném používání aplikace v testovacím období. 0 = velmi nízká, 20 = velmi vysoká. **Pozor**, jedna z dimenzí (C.4) má obrácenou škálu — je to napsané přímo v otázce.

| # | Dimenze | Otázka | Hodnota (0–20) |
|---|---|---|:-:|
| B.1 | **Mentální zátěž** | Jak mentálně náročné bylo zaznamenat jídlo nebo aktivitu? (přemýšlení, rozhodování, hledání položek) | |
| B.2 | **Časová zátěž** | Jak silný časový tlak jsi pociťoval/a při zaznamenávání? (např. že musíš stihnout zaznamenat jídlo, než zapomeneš, nebo že tě to zdržuje od jiné činnosti) | |
| B.3 | **Úsilí** | Jak velké úsilí (mentální i fyzické) jsi musel/a celkově vynaložit, abys dostal/a do aplikace to, co jsi chtěl/a? | |
| B.4 | **Vlastní výkon** *(obrácená škála)* | Jak nespokojený/á jsi byl/a s tím, jak se ti dařilo zaznamenat to, co jsi chtěl/a? **Pozor: 0 = velmi spokojen, 20 = velmi nespokojen.** | |
| B.5 | **Frustrace** | Jak silně jsi se cítil/a zmaten/a, podrážděn/a nebo otráven/a? | |
